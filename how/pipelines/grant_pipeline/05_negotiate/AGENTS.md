# Stage 5: Award Negotiation

## Purpose

Finalize grant terms, sign the agreement, and set up compliance infrastructure. This is where many new programs stumble — celebrating the award without building the systems to manage it.

## Processing Instructions

1. Review grant agreement terms (amount, period, reporting schedule, restrictions)
2. **Human gate**: User must review and approve grant agreement before signing
3. Verify terms align with CC's three non-negotiables:
   - Community governance is non-negotiable (funder does not acquire programmatic direction)
   - No data extraction (community context graphs belong to communities)
   - Open standards (no exclusive rights to aDNA or Lattice Protocol)
4. Set up compliance infrastructure:
   - Create budget tracking codes for this specific funder
   - Set reporting calendar with reminders
   - Brief program staff on deliverables and data collection requirements
   - Set up time tracking for grant-funded staff

## Quality Gate (Exit Criteria)

- [ ] Grant agreement reviewed — no conflicts with CC's three non-negotiables
- [ ] **User has approved** the grant agreement
- [ ] Budget tracking codes created for this funder
- [ ] Reporting calendar set with dates and responsible parties
- [ ] Program staff briefed on deliverables and data collection
- [ ] Partnership entity created at `what/partnerships/` if appropriate

## Advancement

Move file to `06_implement/`, update `stage: implement` when agreement is signed and compliance infrastructure is confirmed.
