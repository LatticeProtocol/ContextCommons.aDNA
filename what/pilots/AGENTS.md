# what/pilots/ — Agent Guide

## What's Here

Pilot program designs and evaluation records — detailed plans for each pilot community including timeline, logistics, hardware requirements, mentor assignments, curriculum localization, success metrics, and evaluation results. One file per pilot.

## Working Rules

- **Naming**: `pilot_{descriptive_name}.md` (underscores, lowercase)
- **Required frontmatter**: `type: pilot`, `created`, `updated`, `status`, `last_edited_by`, `tags`, `community`, `launch_target`, `phase`, `mentor_count`, `builder_count`
- **Check before overwriting**: Read `updated` field — if edited today by someone else, confirm with user
- **Set audit trail**: Update `last_edited_by` and `updated` on every edit
- Pilots are sovereign — each adapts the curriculum to its own context (Standing Order #9). Hardware and logistics are first-class concerns (Standing Order #12).

## Load/Skip Decision

**Load this directory when**:
Planning pilot launches, evaluating outcomes, preparing community onboarding kits.

**Skip when**:
Working on curriculum content that is not pilot-specific.

**Token cost**: ~300 tokens (this AGENTS.md). Individual records are ~200-400 tokens each.
