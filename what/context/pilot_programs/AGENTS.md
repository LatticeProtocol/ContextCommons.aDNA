# AGENTS.md — Pilot Programs Context Topic

## Purpose

Context files defining principles, evaluation methods, and logistics for Context Commons pilot deployments. These are program-level design documents, not community-specific plans — individual pilot plans live in `what/pilots/`.

## Contents

| File | Description | Status |
|------|-------------|--------|
| `context_cc_pilot_design_principles.md` | Sovereignty, adaptability, evaluation, governance principles for pilot design | Active |
| `context_cc_pilot_evaluation_framework.md` | Success metrics, data collection, reporting cadence, research ethics | Active |
| `context_cc_pilot_logistics.md` | Hardware, space, connectivity, scheduling, budget template, pre-launch checklist | Active |

## Loading Strategy

- **Full topic**: ~8K tokens — load all 3 files for pilot planning work
- **Evaluation only**: Load `context_cc_pilot_evaluation_framework.md` (~3K) for assessment/reporting tasks
- **Logistics only**: Load `context_cc_pilot_logistics.md` (~3K) for budget/hardware/scheduling tasks

## Key Dependencies

- `what/context/program_design/` — thesis, architecture, governance model (foundational)
- `what/context/community_engagement/` — partnership model, community requirements, mentor roles
- `what/context/curriculum_framework/` — track structures and timelines referenced in scheduling
- `who/communities/` — pilot community profiles with site-specific infrastructure
- `what/context/fundraising/` — budget context for logistics
- `what/pilots/` — community-specific pilot plans that operationalize these principles
